## 039 [소프트웨어] 소프트웨어의 계층 구조

---

### 소프트웨어 계층
지질학의 지층과 비슷하며, 계층은 서로의 관심사를 구분 짓는다. 각 계층은 맡은 기능을 구현하고, 상위 계층이 서비스에 접근할 수 있도록 추상화를 제공한다.
1. 하드웨어
   - 가장 밑바닥에 있는 계층
   - 시스템 실행 중 거의 변경이 불가능하다.
     - 시스템 실행 중에도 버스를 이용해 장치를 추가, 제거하는 것은 가능하다.
2. 운영체제
   - 흔히 '커널'이라고 부른다. 
   - 하드웨어 종류와 무관하게 운영체제는 하드웨어의 특수성을 숨기고 애플리케이션에 동일한 인터페이스로 제공한다.
     - 이로 인해 소프트웨어는 하드웨어와 무관하게 독립적으로 작동할 수 있다. 소프트웨어는 CPU가 인텔인지, AMD인지 거의 신경 쓸 필요가 없다.
   - 애플리케이션이 서로의 동작에 간섭하지 않도록 관리한다.
     - 메모리 관리: 프로그램이 실행되는 동안 RAM의 어느 위치에 프로그램을 배치할지 결정한다.
     - 파일 시스템 관리: 보조 기억 장치에 정보를 저장할 위치를 결정한다.
     - 주변 장치 제어: 프린터, 디스플레이 등 주변 장치를 조정하여 애플리케이션 간 충돌을 방지한다.
     - 프로세서 제어: 프로세서를 효율적으로 관리하여 시스템의 모든 기능이 원활하게 작동하도록 보장한다.
3. 라이브러리 집합
   - 프로그래머 개개인이 기능을 새로 만들 필요 없도록 일반적으로 유용한 서비스를 제공한다.
4. 애플리케이션

가끔은 커널, 라이브러리, 애플리케이션 간의 구분이 명확하지 않다. 소프트웨어 구성 요소를 만들고 연결하는 방법에는 경우의 수가 많기 때문이다.
- ex) 커널이 대부분의 일을 처리하고 라이브러리에 덜 의존하거나 그 반대의 상황도 존재한다.
