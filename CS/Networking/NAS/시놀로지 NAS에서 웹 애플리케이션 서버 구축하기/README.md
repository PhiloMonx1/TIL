# 시놀로지 NAS에서 웹 애플리케이션 서버 구축하기

이전 회사에서 [NAS](../)에 PHP 기반 웹 애플케이션 서버를 구축한 경험이 있다.

### 개요
당시 회사에는 배포 프로세스가 존재하지 않고, 호스팅 되고 있는 실서버에서 코드를 변경하여 바로 적용시키는 방식으로 개발이 되고 있었다. 

나는 즉시 로컬 개발 환경을 구축하고 형상관리를 시작했으나, 개발 이후 변경점을 보고할 때 내가 일일히 변경 사항에 대한 프레젠테이션을 해야 했고, 내가 발견하지 못한 에러는 고스란히 실서버에 적용되는 일이 비일비재했다.

QC 프로세스의 필요성을 느낀 나는 DEV 서버를 구축해서 회사 내부에서 다같이 QC를 하길 원했고, 비용이 발생하지 않는 DEV 서버 구축 방법은 회사 NAS를 사용하는 것이었다.

다행히 회사 NAS가 시놀로지 였기 때문에 해당 작업을 지원하는 애플리케이션이 있었고, 편하게 환경을 구성할 수 있었다.

### 환경 구축
1. 웹 서버 패키지 설치
   1. Apache HTTP Server
   2. PHP
   3. MariaDB
2. DB 마이그레이션
3. 가상 호스트 설정
   - 시놀로지의 'Web Station'을 사용해서 특정 포트에 가상 호스트를 설정할 수 있다.
